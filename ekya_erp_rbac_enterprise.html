Campus distribution
Agents are automated system actors — they hold roles just like humans but act on behalf of processes (notifications, sync, reports). They are scoped and audited identically to human users.
Agent permission model
Agents use service accounts with JWT tokens. Tokens expire every 24h. Each agent has a whitelist of exact API endpoints it may call — no wildcard access. All agent actions are written to the audit log with agent_id + triggered_by context.
Agents never hold roles above Coordinator level. Any action requiring HoS or above approval must be queued as a human-approval workflow — the agent cannot self-approve.
Teaching sub-tags
Sub-tags are stored as a staff_subtype enum on the user record. They narrow permissions inside a role — they never grant extra access beyond the base role.
Non-teaching staff sub-tags
Non-teaching roles follow the same pattern. Each designation maps to one system role + one sub-tag. This covers all current and future hires without creating new roles.
